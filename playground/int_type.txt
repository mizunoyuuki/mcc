暗黙の変数定義を廃止して、intというキーワードを導入する。

今までは変数や関数の返り値は全て暗黙のうちにintということになっていた。
したがってわざわざint x;というように変数をその型名と一緒に定義することはせず、新しい識別子は全て新しい変数名だとみなしていた。

今後はそのような仮定を置くことはしない。


・int x;という形で変数を定義するようにする。
・int x = 3;といった初期化式はまだサポートしなくていい。
同様に、int x,y;といったものもまだサポートしなくていい。

・関数は今までは、foo(x,y)といった形で書いていましたが、これをint foo(int x, int y)といった形になるように改造する。

現状、トップレベルは関数定義しかないので、パーサはまず、intを読み、そのあとは必ず関数名のはずなのでそれを読み、
次に int <引数>という列を読む。
