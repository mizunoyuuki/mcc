以前の章では、変数名を一文字にして、aからzまでのローカル変数が常に存在しているものとして、扱うようにしていた。

この説では、１文字より長い名前を持つ識別子をサポートさせる。

```
foo = 1;
bar = 2 + 3;
return foo + bar;
```

こんなコードをコンパイルできるようにしたい。

したがって、パーサーは、識別子一つ一つについて今まで見たことがあるかどうかを判定して、新たなものであれば自動的にスタック領域に変数を割り当てる必要がある。
===>>  確かに！！！！

トークナイザを変更して、複数の文字からなる識別子をTK_IDENT型のトークンとして読み込むように。

変数は連結リストで表す。
LVarという構造体で一つの変数を表す。
先頭の要素をlocalsというポインタで持つことにする。

typedef struct LVar LVar;

struct LVar {
    LVar *next;  // 次の変数 or NULL
    char *name;  // 変数の名前
    int len;     // 名前の長さ
    int offset;  // RBPからのオフセット
};

LVar *locals;


パーサにおいては、TK_IDENT型のトークンが出現した場合、その識別子が今までに出現したことがあるかどうかを確認する。
localsをたどって変数名を見ていくことで、既存の変数かどうかはわかる。変数が以前に出現していた場合、その変数のoffsetをそのまま使う。新たな変数の場合は、新しいLVarを作って、新たなオフセットをセットし、そのオフセットをRBPから使ってストアする。
