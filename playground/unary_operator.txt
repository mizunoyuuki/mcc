ポインタを実装する最初のステップとして、アドレスを返す単項＆と、アドレスを参照する単項*を実装する。

これらの演算子は本来はポインタ型の値を返したり、ポインタ型の値をとったりする演算子ですが、現状のコンパイラには整数以外の型がない。
ポインタ型は整数型で代用する。

すなわち、&xは変数xのアドレスを単なる整数として返す、また*xはxの値をアドレスとみなして、そのアドレスから値を読んでくるという演算をする。

動かしたいコード
```
x = 3;
y = &x;
return *y;  // 3を返す
```


また、ローカル変数がメモリ上に連続して割り当てられているということを利用して、スタック上の変数にポインタ型ゆで間接的に無理やりアクセスすることもできるようになる。

```
x = 3;
y = 5;
z = &y + 8;
return *z; // 3を返す
```

実装は比較的簡単らしい。

単項&と単項*を追加した文法は、
```
unary = "+"? primary
      | "-"? primary
      | "*" unary
      | "&" unary
```
		
となるらしい。

### パーサー
トークン列をAST(抽象構文木)に変換する。手法は、再帰下降構文解析(recursive descent parsing)。

演算子の優先順位 = 関数の呼び出し深度

トリビア　再帰下降構文分析の歴史と競合手法

再帰下降構文解析は、1961年にNiklaus Wirth(後のPascalの設計者)らが発表しました。当時のコンパイラ研究は「LR法」「LALR法」などのボトムアップ手法が主流で、再帰下降は「素朴すぎる」と軽視されていた。
しかしGCC、Clang, V8, Rustなど現代の主要コンパイラのほとんどが再帰下降を採用している。
理由は、「エラーメッセージが出しやすい」「デバックしやすい」らしい。
