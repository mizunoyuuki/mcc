構造体(struct)の実装

構造体の本質「連続したメモリ領域に名前をつける」

Cの構造体は、CSの用語では product type(直積型)と呼ばれる。
型Aと型Bの直積 A * B は「Aの値1つとBの値1つのくみ」であり、
Cのstruct {A a; B b;}はこれをメモリ上の連続領域として表現したもの

重要な性質
- メンバは宣言順にメモリに並ぶ - C規格で保証されている
- 各メンバはベースアドレスからの固定オフセットを持つ
- 構造体自体にサイズとアライメントがある。

struct Foo {
  char c;    // offset 0
  int i;     // offset 4  (3バイトのパディングがcの後に入る)
  char *p;   // offset 8
};

sizeof(Foo) => 16
align = 8

末尾のアライメントも必要になる。
これは、構造体配列 struct Foo arr[10]で、
各要素のアライメントが崩れないように最大サイズも最大アラインメントの倍数に切り上げる必要がある。

Cの型システムには４つの独立した名前空間(namespace)がある。

1. ラベル名  : gotoの飛び先
2. タグ名    : struct, union, enumの名前
3. メンバ名  : 各構造体/共用体の中のメンバ
4. 通常識別子: 変数名、関数名、typedef名, enum定数

この名前空間とは、シンボルテーブルで衝突しない単位ってこと

→ようは、
- struct Foo; と int Foo;  は共存可能(Fooはタグ、Fooは通常識別子)
- 同じ構造体にxというメンバがあっても、別の構造体のxメンバと衝突しない。
- typede struct Foo Foo; とすると、タグのFooとtypedefのFooが両方存在する


C言語の不完全型(imcomplete type)

struct A x; とかけるためにはstruct Aの定義がすでに見えている必要がある。
struct A *p;

のようにポインタであれば、定義が見えなくても宣言できる。
これはC言語の「不完全型」という概念で、サイズが未定でもポインタのサイズは既知だから。
